第5周周报

一、项目实践内容
1. DOM 元素操作
本周主要进行了 DOM（文档对象模型）的操作练习。通过 `document.getElementById` 和 `querySelector` 等方法获取页面元素，并尝试通过 JavaScript 修改元素的样式和内容，实现了页面内容的动态更新。

2. 事件监听机制
制作了一个简易的“待办事项”列表功能。实现了输入框的回车事件监听，动态创建列表项并添加到页面中。在处理列表项点击事件时，学习并应用了“事件委托”技术，将事件绑定在父元素上，提高了程序性能。

3. 本地存储应用
为了解决页面刷新后数据丢失的问题，尝试使用了浏览器提供的 `localStorage` 功能。实现了将数据序列化为 JSON 字符串进行存储，以及读取时进行解析还原的流程。

二、实践收获
1. 掌握了 DOM 操作的基本方法，但同时也体会到了频繁操作 DOM 对代码整洁度的影响。
2. 理解了事件冒泡机制，并学会了利用这一特性实现事件委托，简化了动态元素的事件绑定处理。
3. 熟悉了 Web Storage API 的基本用法，了解了前端本地存储在保持用户状态方面的应用。

三、实践体会
本周实现了真正的页面交互功能，使网页不再是静态的展示。但在操作 DOM 的过程中，发现代码量迅速膨胀，逻辑变得复杂。通过查阅资料，了解到现代前端框架可以有效解决这一问题，对后续的学习充满了期待。

四、建议
在进行 DOM 操作前，建议先对获取的元素进行非空判断，以避免因元素不存在而导致的运行错误。此外，在使用 `localStorage` 存储复杂数据时，务必记得进行 JSON 序列化处理。
