第14周周报

一、项目实践内容
1. 比赛开始页面开发
本周完成了 `MatchStart.vue` 页面的开发。实现了从后端 API 获取选手列表并填充到下拉框的功能。使用了 Vue 的 `v-model` 实现表单数据的双向绑定，并添加了简单的表单验证，确保用户必须选择完所有必填项才能点击“开始比赛”。

2. 比赛控制页面布局
完成了 `MatchControl.vue` 的基础布局开发。设计了比分展示面板，分别显示两名选手的当前得分。添加了“得分”、“结束比赛”、“导出 Excel”等操作按钮。使用了 Flexbox 布局确保界面在不同窗口大小下都能保持良好的视觉效果。

3. 路由跳转实现
配置了 Vue Router，实现了从比赛开始页面跳转到比赛控制页面的逻辑。在跳转时，通过路由参数传递了比赛 ID，确保控制页面能够加载正确的比赛数据。

二、实践收获
1. 熟练掌握了 Vue 的表单处理技巧，包括 `<select>` 下拉框的数据绑定和联动（如选择选手后，发球方列表更新）。
2. 学会了使用 Vue Router 的 `useRouter` 和 `useRoute` 组合式 API 进行编程式导航和参数获取。
3. 掌握了 Axios 的基本封装技巧，创建了统一的 API 实例，设置了 BaseURL 和超时时间，减少了重复代码。

三、实践体会
在开发比赛控制页面时，我意识到状态管理的重要性。比分、回合数、当前发球方等状态需要在页面上实时更新，这需要对 Vue 的响应式系统有深入的理解。虽然页面看起来不复杂，但背后的数据流转逻辑还是挺绕的。

四、建议
在处理异步请求（如获取选手列表）时，建议加上 `loading` 状态和错误处理逻辑，这样在网络较慢或请求失败时，用户能看到提示，而不是对着空白页面发呆。
