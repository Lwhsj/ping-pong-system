第12周周报

一、项目实践内容
1. Electron 与 Vue 技术整合
本周完成了 Vue 框架与 Electron 的整合工作。选用了 Vite 作为构建工具，利用其快速构建的优势提升开发体验。解决了开发环境与生产环境下路径引用不一致的问题，确保了应用在不同环境下的正常运行。

2. 项目初始化与结构搭建
创建了网球系统客户端的项目仓库，按照规范搭建了目录结构。将 Vue 页面代码放置于渲染进程目录，Electron 核心代码放置于主进程目录，并配置了基础的代码规范。

3. 自定义界面实现
为了提升应用美观度，隐藏了系统原生的标题栏，并使用 Vue 组件实现了自定义标题栏。通过 IPC 通信实现了自定义标题栏对窗口最小化、最大化和关闭操作的控制。

二、实践收获
1. 掌握了 Electron + Vue + Vite 技术栈的整合流程，体验到了现代化构建工具带来的高效开发流程。
2. 解决了打包后资源路径失效的常见问题，理解了 Web 应用与桌面应用在资源加载方式上的差异。
3. 学习了 CSS 在桌面应用中的特殊属性（如窗口拖拽区域设置），实现了更符合桌面应用交互习惯的界面效果。

三、实践体会
技术整合阶段涉及较多配置工作，容易出现环境兼容性问题。通过查阅文档和社区资料，逐一解决了 Node 模块加载、路径配置等问题。环境搭建完成后，能够专注于业务逻辑的开发，感受到了技术栈融合带来的便利。

四、建议
在进行技术选型和环境搭建时，优先考虑使用社区成熟的脚手架工具，这能避免大量的繁琐配置。同时，深入理解构建工具（如 Vite）的基本原理，对于解决打包部署问题大有裨益。
