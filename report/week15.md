第15周周报

一、项目实践内容
1. 核心计分逻辑实现
本周重点攻克了比赛计分逻辑。在 `MatchControl.vue` 中实现了“得分”按钮的点击事件处理。每次得分后，前端先更新本地显示的临时比分，然后调用 `recordRally` 接口将本回合数据同步给后端，确保数据的一致性。

2. 视频录制功能开发
利用 Electron 提供的能力，开发了 `useVideoRecorder.js` 组合式函数（Composable）。实现了调用系统摄像头 (`navigator.mediaDevices.getUserMedia`)，并使用 `MediaRecorder` API 进行视频录制。录制结束后，将视频数据封装为 Blob 对象，准备上传。

3. 交互优化
优化了页面交互体验。添加了加载中的 Loading 状态提示，防止用户重复点击按钮。在比赛结束时增加了确认提示，避免误操作。

二、实践收获
1. 深入理解了浏览器的多媒体 API，成功实现了纯前端的视频录制功能，这在以前觉得是非常复杂的技术。
2. 掌握了 Vue 3 的 Composition API 写法，将视频录制逻辑抽离成独立的 `useVideoRecorder` Hook，使得组件代码更加简洁，逻辑复用性更强。
3. 学会了处理文件上传，了解了 `FormData` 对象的使用，它是前端发送二进制文件（如视频、图片）的标准方式。

三、实践体会
视频录制功能开发过程中遇到了不少坑，比如不同浏览器对视频格式的支持不同（WebM vs MP4）。最终选择了兼容性较好的 WebM 格式。此外，录制长视频会导致内存占用增加，需要注意及时释放资源。

四、建议
在开发涉及硬件调用（如摄像头）的功能时，一定要注意权限处理。Electron 需要在主进程或系统层面配置相应的权限，否则无法获取媒体流。
