第11周周报

一、项目实践内容
1. 进程间通信（IPC）实践
本周主要攻克了 Electron 开发中的核心难点——进程间通信。实现了一个系统信息获取功能：渲染进程通过 IPC 请求，主进程调用 Node.js 接口获取 CPU 信息并返回。使用了 `ipcRenderer.invoke` 和 `ipcMain.handle` 的异步通信模式。

2. 预加载脚本（Preload）应用
出于安全性考虑，学习了预加载脚本的使用。将 IPC 通信逻辑封装在 `preload.js` 中，并通过 `contextBridge` 将受限的 API 暴露给渲染进程，实现了渲染进程与 Node.js 环境的安全隔离。

3. 调试技巧掌握
在开发过程中，学习了如何同时调试主进程和渲染进程。掌握了查看主进程终端日志和渲染进程开发者工具（DevTools）的方法，有效提高了问题排查效率。

二、实践收获
1. 深入理解了 IPC 通信机制，掌握了“请求-响应”式的异步通信模式，这与前后端接口调用的逻辑有异曲同工之妙。
2. 认识到预加载脚本在 Electron 安全架构中的重要地位，它是连接渲染进程与系统底层的安全桥梁。
3. 掌握了 Electron 应用的调试方法，能够快速定位通信失败或代码执行错误的原因。

三、实践体会
IPC 通信的代码编写相对繁琐，增加一个功能往往涉及主进程、预加载脚本和渲染进程三处的修改。但为了保障应用的安全性和架构的清晰度，这种设计是必要的。初期常因忘记在 Preload 中暴露接口而导致调用失败，通过练习逐渐熟悉了这一流程。

四、建议
在定义 IPC 通信频道名称（Channel Name）时，建议使用常量进行统一管理，避免因手动输入字符串导致的拼写错误，从而减少不必要的调试时间。
